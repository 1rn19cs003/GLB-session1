⚙️ Middleware Pipeline
Request Flow:
Client Request → Express Server → Route Matching → Middleware Stack → Controller Logic → Response
🔐
auth.js - JWT Verification
✓ Validates JWT token from Authorization header
✓ Decodes token and extracts user information
✓ Attaches user object to request (req.user)
✓ Rejects request if token is invalid or expired
👤
rbac.js - Role-Based Access Control
✓ Checks user role (admin, user, moderator, etc.)
✓ Authorizes access based on role permissions
✓ Ensures users can only perform allowed actions
✓ Returns 403 Forbidden if access denied

🔐 Authentication & Authorization Flow
Login Flow:
1. User submits email/password via POST /auth/login
2. authController validates credentials against users.json
3. If valid, JWT token is generated with user ID and role
4. Token sent to client and stored (localStorage/cookie)
5. Client includes token in Authorization header for requests
Protected Route Flow:
1. Client sends request with JWT in Authorization header
2. auth.js middleware verifies JWT signature and expiry
3. rbac.js checks user role for endpoint permissions
4. If authorized, controller processes request
5. Response sent with updated data or confirmation
